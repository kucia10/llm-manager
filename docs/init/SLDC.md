다음은 **코드 어시스턴트(Code Assistant) 구축을 위한 SDLC 전반 시나리오**에서
**RFP → 설계 → 개발 → 테스트 → 배포 → 운영 → 매뉴얼**까지 단계별로 **작성해야 할 항목 목록**을 체계적으로 정리한 것이다.
(실제 문서 목차로 바로 전환 가능하도록 구성)

---

## 1. RFP / 제안요청 단계

### 1.1 사업 개요

* 사업 목적 및 배경
* 코드 어시스턴트 도입 필요성
* 기대 효과 (생산성, 품질, 비용, 리스크 감소)
* 적용 범위 (조직, 프로젝트, 언어, 플랫폼)

### 1.2 현황 분석

* 기존 개발 프로세스 현황
* 기존 IDE / CI/CD / 형상관리 환경
* 기존 AI/자동화 도구 사용 여부
* 문제점 및 개선 요구사항

### 1.3 요구사항 정의

#### 기능 요구사항

* 코드 생성 (Boilerplate / 비즈니스 로직)
* 코드 설명 / 주석 생성
* 리팩토링 제안
* 오류 분석 및 수정 제안
* 테스트 코드 생성
* API / SDK 사용 가이드 자동화
* 사내 코드/문서 기반 RAG 지원
* 멀티 언어 / 멀티 프레임워크 지원

#### 비기능 요구사항

* 성능 (응답 시간, TPS)
* 보안 (소스코드 유출 방지, 접근제어)
* 확장성 (모델 교체, 플러그인 구조)
* 가용성 / 장애 대응
* 감사 및 로깅

### 1.4 기술 요구사항

* LLM 아키텍처 요구
* 온프레미스 / 클라우드 여부
* IDE 연동 방식 (VSCode, IntelliJ 등)
* API 방식 (REST / Streaming)
* 모델 추론 방식

### 1.5 사업 관리 요구

* 일정 / 산출물
* 인력 구성
* 유지보수 / SLA
* 검수 기준
